## <b>ADR 001:</b> Definição da tecnologia de banco de dados para o EduVerse

<b>Status:</b> Aceito

<b>Contexto:</b> A definição da tecnologia de um banco de dados (SQL ou NoSQL), muda a forma como um sistema é estruturado e afeta diretamente escalabilidade e flexibilidade, pois cada abordagem possui ganhos e perdas nesses aspectos. Essa decisão é importante nesse momento pois será a base para a concepção da estrutura e escolha do banco de dados (MySQL, MongoDB, etc.).

<b>Decisão:</b> Foi decidida a utilização de ambos SQL e NoSQL no projeto, sendo SQL para os dados principais da aplicação e NoSQL para outros dados em grandes quantidades (logs, cache, etc.). Essa abordagem permite o equilíbrio dos benefícios das duas tecnologias, como a consistência de dados oferecida por SQL e a performance  oferecida por NoSQL.

<b>Consequências (Trade-offs):</b>

<b>Ganhos:</b>

<b>[+]</b> Estrutura baseada em tabelas e relacionamentos para dados de usuários, conteúdos, categorias, entre outros dados críticos com SQL.

<b>[+]</b> Armazenamento e desempenho eficientes para cache, logs e outros dados com NoSQL.

<b>Perdas:</b>

<b>[-]</b> Curva de aprendizado e dificuldade maior para a equipe de desenvolvimento, ao utilizar ambas as abordagens.

<b>[-]</b> Necessidade de integração entre as duas tecnologias.

<b>Alternativas Consideradas:</b> A utilização apenas de bancos de dados SQl para todos os dados da aplicação foi considerada, porém descartada devido à possibilidade de grande volume de dados gerados pelo sistema, como logs de compliance e dados de cache por usuário. Esse cenário poderia exigir maior desempenho em operações de escrita e armazenamento eficiente, características que podem ser melhor atendidas por bancos NoSQL.
